# 如何重新运行修改后的应用

## 🚀 方法1: 在Android Studio中运行（推荐）

### 步骤1: 同步Gradle依赖
1. 点击顶部菜单：**File** → **Sync Project with Gradle Files**
   - 或者点击工具栏的 **🐘 Gradle图标** → **Sync Now**
   - 或者使用快捷键：`Ctrl + Shift + O` (Windows) 或 `Cmd + Shift + O` (Mac)

### 步骤2: 清理构建（可选，但推荐）
1. 点击顶部菜单：**Build** → **Clean Project**
   - 这会清理之前的构建缓存
   - 等待清理完成（通常几秒钟）

### 步骤3: 重新构建项目
1. 点击顶部菜单：**Build** → **Rebuild Project**
   - 这会重新编译所有代码
   - 等待构建完成（可能需要1-3分钟）

### 步骤4: 运行应用
1. **连接设备**：
   - 用USB线连接Android手机
   - 在手机上启用"USB调试"
   - 或者使用Android模拟器

2. **选择运行配置**：
   - 在工具栏顶部，选择设备（你的手机或模拟器）
   - 确保选择了正确的构建变体（通常是 `debug`）

3. **点击运行按钮**：
   - 点击工具栏的 **▶️ Run** 按钮
   - 或使用快捷键：`Shift + F10` (Windows) 或 `Ctrl + R` (Mac)
   - 或点击菜单：**Run** → **Run 'app'**

4. **等待安装**：
   - Android Studio会自动：
     - 编译代码
     - 打包APK
     - 安装到设备
     - 启动应用

## 🔧 方法2: 使用Gradle命令（命令行）

### 在项目根目录执行：

#### Windows (PowerShell):
```powershell
# 1. 清理构建
.\gradlew clean

# 2. 构建Debug版本
.\gradlew assembleDebug

# 3. 安装到设备
.\gradlew installDebug
```

#### Mac/Linux:
```bash
# 1. 清理构建
./gradlew clean

# 2. 构建Debug版本
./gradlew assembleDebug

# 3. 安装到设备
./gradlew installDebug
```

## 📱 方法3: 直接安装APK（如果已经构建过）

### 步骤1: 找到APK文件
APK文件位置：
```
app/build/outputs/apk/debug/app-debug.apk
```

### 步骤2: 安装到设备
1. **通过USB**：
   ```powershell
   # 连接设备后，使用adb安装
   adb install app\build\outputs\apk\debug\app-debug.apk
   ```

2. **手动安装**：
   - 将APK文件复制到手机
   - 在手机上打开文件管理器
   - 点击APK文件安装
   - 允许"未知来源"安装（如果提示）

## ⚡ 快速运行（最快方法）

### 在Android Studio中：
1. **连接设备**（手机或模拟器）
2. **点击运行按钮** ▶️
   - Android Studio会自动检测代码变化
   - 如果只是UI代码修改，通常只需要重新运行即可
   - 不需要每次都Clean和Rebuild

### 快捷键：
- **运行**: `Shift + F10` (Windows) 或 `Ctrl + R` (Mac)
- **停止**: `Ctrl + F2` (Windows) 或 `Cmd + F2` (Mac)

## 🔍 常见问题

### 问题1: "Gradle sync failed"
**解决**：
- 检查网络连接（需要下载依赖）
- 点击 **File** → **Invalidate Caches / Restart** → **Invalidate and Restart**
- 重新同步：**File** → **Sync Project with Gradle Files**

### 问题2: "Installation failed"
**解决**：
- 确认设备已连接：`adb devices`
- 检查设备是否启用了USB调试
- 尝试卸载旧版本：`adb uninstall com.edusmart.app`
- 然后重新安装

### 问题3: "App keeps crashing"
**解决**：
- 查看Logcat中的错误信息
- 检查是否有编译错误
- 确认所有依赖都已正确同步

### 问题4: "Changes not reflected"
**解决**：
- 执行 **Build** → **Clean Project**
- 然后 **Build** → **Rebuild Project**
- 或者卸载应用后重新安装

## 📊 构建变体说明

### Debug版本（开发用）
- 位置：`app/build/outputs/apk/debug/app-debug.apk`
- 特点：未优化，包含调试信息，构建快
- 用途：日常开发和测试

### Release版本（发布用）
- 位置：`app/build/outputs/apk/release/app-release.apk`
- 特点：已优化，代码混淆，体积小
- 用途：正式发布

## 🎯 推荐工作流程

1. **修改代码** ✅
2. **保存文件** (`Ctrl + S`)
3. **点击运行** ▶️
   - Android Studio会自动：
     - 检测变化
     - 增量编译
     - 安装更新
     - 重启应用

**注意**：如果修改了：
- 依赖配置（`build.gradle.kts`）→ 需要同步Gradle
- 资源文件（布局、图片等）→ 可能需要Clean
- 代码签名配置 → 需要Rebuild

---

## 💡 小贴士

1. **增量编译**：Android Studio会智能地只重新编译修改的文件，所以通常直接运行就够快

2. **热重载**：对于Compose UI，某些情况下可以热重载，无需完全重启

3. **构建缓存**：如果遇到奇怪的问题，可以清理缓存：
   - **File** → **Invalidate Caches / Restart**

4. **查看构建日志**：在Android Studio底部的 **Build** 标签页查看详细构建信息
