## 🔄 Git 分支管理策略

### 主分支结构

```
main (master)
├── develop (开发主分支)
│   ├── feature/scan-ocr (拍照识题功能)
│   ├── feature/ar-space (AR功能)
│   ├── feature/smart-note (智能笔记)
│   ├── feature/speaking (口语私教)
│   ├── feature/radar (知识雷达)
│   └── feature/ui-components (UI组件)
```

### 分支命名规范

- `feature/模块名-功能` - 功能开发分支
- `fix/模块名-问题` - Bug修复分支
- `refactor/模块名` - 重构分支
- `docs/文档类型` - 文档更新

### 工作流程

1. **从 develop 创建功能分支**
   ```bash
   git checkout develop
   git pull origin develop
   git checkout -b feature/scan-ocr
   ```

2. **开发并提交**
   ```bash
   git add .
   git commit -m "feat: 添加OCR识别功能"
   git push origin feature/scan-ocr
   ```

3. **合并到 develop**
   - 创建 Pull Request
   - 代码审查
   - 合并到 develop

4. **发布到 main**
   - 测试通过后
   - 合并 develop 到 main
   - 打标签发布

## 📝 开发流程步骤

### 阶段1: 环境准备（所有人）

1. **克隆项目**
   ```bash
   git clone <项目地址>
   cd 软件创新大赛
   ```

2. **创建开发分支**
   ```bash
   git checkout -b develop
   git checkout -b feature/你的模块名
   ```

3. **同步依赖**
   - 在 Android Studio 中：File → Sync Project with Gradle Files

### 阶段2: 模块开发（各自负责）

#### 每个开发者的工作流程：

1. **理解需求**
   - 阅读模块相关代码
   - 理解现有实现
   - 明确需要实现的功能

2. **设计实现**
   - 规划代码结构
   - 定义接口和数据结构
   - 设计UI界面

3. **编码实现**
   - 编写代码
   - 添加注释
   - 遵循代码规范

4. **本地测试**
   - 运行应用
   - 测试功能
   - 修复Bug

5. **提交代码**
   ```bash
   git add .
   git commit -m "feat: 实现XX功能"
   git push origin feature/你的分支名
   ```

### 阶段3: 代码审查和合并

1. **创建 Pull Request**
   - 在 Git 平台（GitHub/GitLab）创建 PR
   - 描述实现的功能
   - 标记相关开发者审查

2. **代码审查**
   - 其他开发者审查代码
   - 提出修改建议
   - 讨论实现方案

3. **合并到 develop**
   - 审查通过后合并
   - 解决可能的冲突
   - 更新文档

### 阶段4: 集成测试

1. **合并所有功能到 develop**
2. **整体测试**
   - 功能测试
   - 兼容性测试
   - 性能测试

3. **修复集成问题**
   - 解决模块间冲突
   - 统一接口规范
   - 优化整体体验

## 🛠️ 开发规范

### 代码规范

1. **命名规范**
   - 类名：大驼峰 `HomeScreen`
   - 函数名：小驼峰 `recognizeText`
   - 常量：大写下划线 `XUNFEI_APP_ID`
   - 文件名：与类名一致

2. **注释规范**
   ```kotlin
   /**
    * 识别图片中的文字
    * @param imagePath 图片文件路径
    * @return 识别出的文字内容
    */
   suspend fun recognizeText(imagePath: String): String
   ```

3. **提交信息规范**
   - `feat: 新功能`
   - `fix: 修复Bug`
   - `refactor: 重构代码`
   - `docs: 更新文档`
   - `style: 代码格式`
   - `test: 添加测试`

### 文件组织

```
app/src/main/java/com/edusmart/app/
├── feature/          # 功能模块
│   ├── scan/         # 拍照识题
│   ├── ar/           # AR知识空间
│   ├── note/         # 智能笔记
│   ├── speaking/     # 口语私教
│   └── radar/        # 知识雷达
├── service/          # 服务层
├── ui/               # UI组件
├── data/             # 数据层
└── config/           # 配置
```

## 🔗 模块间接口定义

### 数据接口

```kotlin
// 题目识别结果
data class QuestionRecognitionResult(
    val content: String,
    val subject: String,
    val knowledgePoints: List<String>
)

// AI答案结果
data class AIAnswerResult(
    val answer: String,
    val explanation: String,
    val steps: List<String>
)
```

### 服务接口

```kotlin
// OCR服务接口
interface IOCRService {
    suspend fun recognizeText(imagePath: String): String
}

// AI服务接口
interface IAIService {
    suspend fun answerQuestion(question: String): AIAnswerResult
}
```
