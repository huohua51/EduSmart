# 修复构建错误指南

## 🔍 错误分析

从构建日志看到的主要问题：
1. **SSL peer shut down incorrectly** - TLS/SSL 握手失败
2. **多个任务失败** - 都是因为依赖解析失败导致的连锁反应
3. **dataBindingMergeDependencyArtifacts** - 数据绑定相关任务失败

## ✅ 已完成的修复

### 1. 禁用数据绑定
- 在 `app/build.gradle.kts` 中设置 `dataBinding = false`
- 避免数据绑定相关的构建错误

### 2. 优化 Gradle 配置
- 添加了并行构建和缓存配置
- 优化了 JVM 参数

### 3. 注释掉问题依赖
- SceneView 依赖已注释
- ARActivity 代码已暂时禁用

## 🚀 解决步骤

### 步骤1: 清理构建缓存

**方法A: 使用批处理脚本（推荐）**
1. 双击运行 `清理构建缓存.bat`
2. 等待清理完成

**方法B: 手动清理**
```powershell
# 删除项目构建文件
Remove-Item -Recurse -Force build, app\build, .gradle -ErrorAction SilentlyContinue

# 清理 Gradle 缓存（可选，会删除所有项目的缓存）
Remove-Item -Recurse -Force $env:USERPROFILE\.gradle\caches -ErrorAction SilentlyContinue
```

### 步骤2: 在 Android Studio 中清理

1. **清理缓存**：
   - 点击：**File** → **Invalidate Caches / Restart**
   - 选择：**Invalidate and Restart**
   - 等待 Android Studio 重启

2. **同步 Gradle**：
   - 点击：**File** → **Sync Project with Gradle Files**
   - 或点击工具栏的 **🐘 Gradle 图标**

3. **清理项目**：
   - 点击：**Build** → **Clean Project**
   - 等待完成

4. **重新构建**：
   - 点击：**Build** → **Rebuild Project**
   - 等待构建完成

### 步骤3: 如果仍然失败

#### 3.1 检查网络连接
- 确保可以访问 Maven Central 和 Google Maven
- 如果在公司网络，可能需要配置代理

#### 3.2 配置代理（如果需要）
在 `gradle.properties` 中添加：
```properties
systemProp.http.proxyHost=your.proxy.host
systemProp.http.proxyPort=8080
systemProp.https.proxyHost=your.proxy.host
systemProp.https.proxyPort=8080
```

#### 3.3 使用离线模式（如果依赖已下载）
```properties
# 在 gradle.properties 中添加
org.gradle.offline=true
```

#### 3.4 使用国内镜像（如果在中国）
在 `settings.gradle.kts` 中添加阿里云镜像：
```kotlin
repositories {
    // 阿里云镜像（国内用户推荐）
    maven { url = uri("https://maven.aliyun.com/repository/public/") }
    maven { url = uri("https://maven.aliyun.com/repository/google/") }
    maven { url = uri("https://maven.aliyun.com/repository/gradle-plugin/") }
    
    google()
    mavenCentral()
    // ...
}
```

## 🔧 常见问题

### Q1: 仍然显示 SSL 错误
**A**: 
- 检查防火墙设置
- 尝试使用 VPN
- 检查系统时间是否正确

### Q2: 依赖下载很慢
**A**: 
- 使用国内镜像（阿里云）
- 配置代理
- 使用离线模式（如果依赖已下载）

### Q3: 某些依赖无法下载
**A**: 
- 检查依赖版本是否正确
- 尝试更新到最新版本
- 检查仓库配置

## 💡 快速修复命令

在项目根目录执行：

```powershell
# 清理并重新构建
.\gradlew clean build --refresh-dependencies

# 或者只清理
.\gradlew clean

# 然后重新同步
# 在 Android Studio 中：File -> Sync Project with Gradle Files
```
